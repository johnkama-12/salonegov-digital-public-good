# SaloneGov Project Management Plan

## Project Duration

The project plan is scheduled from Week 4 to Week 11. It includes requirements analysis, SDLC planning, architecture design, UML diagrams, prototype development, GitHub repository setup, testing, documentation and final presentation.

## Weekly Schedule

| Week | Activity |
|---|---|
| Week 4 | Understand assignment brief and select digital public good idea |
| Week 5 | Gather requirements and define system features |
| Week 6 | Select SDLC model and design system architecture |
| Week 7 | Create use case, activity and sequence diagrams |
| Week 8 | Create class diagram, ERD and data flow diagram |
| Week 9 | Design Figma prototype screens |
| Week 10 | Prepare GitHub repository, README, license and documentation |
| Week 11 | Final testing, review, presentation and submission |

## Git Workflow

- Create the repository on GitHub.
- Use the main branch for stable project files.
- Commit each major change with a clear message.
- Store report, diagrams, prototype link and documentation in organized folders.
- Tag the final submission version as v1.0-final.

## Deliverables

- Project report
- System architecture diagram
- UML diagrams
- ERD and DFD diagrams
- Figma prototype link
- GitHub repository
- README and license
