# SaloneGov Requirements

## Functional Requirements

1. The system shall allow citizens to create an account and log in securely.
2. The system shall allow users to browse government services by category.
3. The system shall allow users to submit service application forms online.
4. The system shall allow users to upload supporting documents.
5. The system shall allow users to track application status.
6. The system shall allow government staff to review submitted applications.
7. The system shall allow government staff to approve or reject applications.
8. The system shall send notifications about application updates.
9. The system shall allow administrators to manage services and users.
10. The system shall store application, user, document and payment records.

## Non-Functional Requirements

1. The system should be simple and user-friendly.
2. The system should be responsive on mobile, tablet and desktop devices.
3. The system should protect user data with secure login and access control.
4. The system should support future integration with government ministries.
5. The system should be maintainable and easy to extend.

## Main Users

- Citizens
- Businesses
- Foreign nationals
- Government staff
- System administrator
